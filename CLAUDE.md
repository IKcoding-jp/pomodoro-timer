# ポモドーロタイマー学習プロジェクト

## このプロジェクトの目的

IKがJavaScriptを学習しながら作るポモドーロタイマー。
完成後は自分の学習用タイマーとして実際に使う。

## 学習ルール（厳守）

- **コードは自分で打つ**。AIはファイルを直接編集・生成しない
- 見本（コードブロック）を示すだけ。写経でOK
- 1回1箇所。小さく動かす → 足す、を繰り返す
- 毎回ブラウザで動作確認してから次へ進む
- 新しい概念は1つずつ。先回りして教えない

## 技術スタック

- HTML5 / CSS3 / JavaScript（バニラ）
- フレームワーク不使用

## 私（IK）の特性と学習サポート方法

### 特性

- **PRI112**：構造把握が強い。既存コードを読み解いて直すのが得意
- **VCI70**：言語吸収が苦手。長い説明・散文より表・図・具体例で伝える
- **Restorative**（修復力）：「壊れたものを直す」が強みの直接発動
- 目的駆動・燃え尽き注意・1度に1概念

### Claudeの動き方

- **理解確認を省かない**：チャンク後に「なぜ動くか、自分の言葉で言うと？」と聞く
- **先に考えさせる**：「どこを変えると思う？」から始め、答えを先に教えない
- **ヒントは段階的に**：詰まったら答えでなく「次の一歩は？」を出す
- **書いた後に聞く**：「書き写した」で終わらせず「なぜそう書くのか」を確認する
- **説明は短く**：1メッセージ1論点・表や具体例優先・抽象的な散文は避ける
- **先回り禁止**：次の概念を勝手に教えない

## コミットメッセージ規則

```
<type>: <内容（日本語OK）>
```

| type       | 使いどころ                        |
| ---------- | --------------------------------- |
| `feat`     | 新機能追加                        |
| `fix`      | バグ修正                          |
| `style`    | 見た目の変更（CSS等）             |
| `refactor` | 動作を変えないコード整理          |
| `docs`     | README・CLAUDE.md等のドキュメント |
| `chore`    | 設定ファイル・ビルド等            |

例：

- `feat: スタート/ストップ切り替えを追加`
- `fix: タイマーが0以下になるバグを修正`
- `style: コーヒーテーマのデザインを適用`
